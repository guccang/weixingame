# 世界观模块

这份文档用于定义当前项目的世界观设定、运行时接入点、存档结构，以及后续扩展时必须遵守的规则，避免页面文案、关卡表现、Boss 事件和结算口径再次分散。

适用范围：

- `js/worldview/index.js`
- `js/game/landmarks.js`
- `js/main.js`
- `js/game/runDirector.js`
- `js/monster/boss.js`
- `js/drawer/startScreen.js`
- `js/drawer/modeSelect.js`
- `js/drawer/gameOver.js`

## 1. 核心设定

当前游戏的统一世界观是：

- 世界名称：`天穹跃迁联赛`
- 赛季名称：`城市空域巡回赛`
- 核心口号：`穿越赛区云层，争夺冠位席次。`

玩家身份不是单纯“爬塔角色”，而是参加跨城市空域联赛的跃迁选手。

现有系统在世界观中的解释如下：

- 主页面：联赛大厅
- 无尽模式：巡回排位赛
- 闯关模式：指定赛区考核
- 竞速模式：限时冲榜赛
- 地标：赛区 / 空域分站
- Boss：赛区守关单位或空投干扰单位
- 特殊平台 / Buff / 事件：赛场机制、补给投放、赛道扰动
- 结算页：赛事战报

## 2. 当前已落地的赛区和身份体系

### 联赛席位

按历史最高分决定玩家当前联赛身份：

- `rookie`：见习跃迁员
- `contender`：外卡挑战者
- `vanguard`：主赛区先锋
- `elite`：空域精英
- `legend`：冠位竞速者

用途：

- 主页面副标题
- 角色预览区身份提示
- 开局播报
- 结算战报身份标签

### 赛区定义

当前赛区与地标的统一映射：

- `greatwall`：长城资格赛区
- `eiffeltower`：巴黎外卡赛区
- `statueofliberty`：海湾主赛区
- `burjkhalifa`：冠位高空赛区

每个赛区至少包含：

- `leagueTitle`：赛区名
- `arenaDesc`：赛道说明
- `bossFactionHint`：守关风格提示
- `eventLabel`：局内赛段名称

规则：

- 新增地标时，不能只配 `name/targetHeight/desc`。
- 必须同时补齐竞技语义字段，否则世界观模块无法给主页、赛段和结算生成统一文案。

### Boss 身份

当前 Boss 在世界观中的身份：

- `leaper`：近压守区者
- `thrower`：空投干扰官
- `default`：赛区守卫

每个 Boss 档案至少要有：

- `title`
- `warningText`
- `spawnText`
- `interruptText`
- `defeatText`

可选补充：

- `missText`
- `hitText`
- `dashText`
- `growthText`
- `exitText`

## 3. 模块职责

世界观文案和进度统一由 `js/worldview/index.js` 提供。

当前职责：

- 定义联赛基础设定
- 定义赛区身份和 Boss 身份
- 根据进度推导联赛席位
- 生成主页面世界观文案
- 生成开局播报
- 生成赛段切换文案
- 生成 Boss 事件文案
- 生成结算战报文案
- 计算世界观解锁结果

规则：

- UI 层不应直接拼接长世界观文本。
- `startScreen`、`runDirector`、`boss`、`gameOver` 只负责消费世界观服务，不负责自行发明新口径。
- 如果一个新页面需要世界观文本，优先在 `worldview` 模块增加查询接口，而不是在页面文件里临时硬编码。

## 4. 当前接入点

### 主页面

接入位置：

- `js/drawer/startScreen.js`

当前接入内容：

- 联赛标题
- 联赛副标题
- 主提示语
- 当前身份提示
- 最近一次解锁提示

规则：

- 主页面标题区必须始终反映联赛语义，不能回退成和局内无关的随机宣传语。
- 如果选择了赛区挑战，主页面提示语要优先反映当前已锁定赛区。

### 赛区选择页

接入位置：

- `js/drawer/modeSelect.js`

当前接入内容：

- 赛区名
- 赛区说明
- 目标高度
- 守关提示

规则：

- 地标面板显示的是“赛区信息”，不是普通景点百科。
- 新赛区卡片必须同时让玩家看懂目标、风格和守关倾向。

### 局内

接入位置：

- `js/main.js`
- `js/game/runDirector.js`
- `js/drawer/ui.js`

当前接入内容：

- 开局播报
- 赛段切换 banner
- 顶部赛段 pill

规则：

- 局内世界观文案必须短、快、可扫读，不能妨碍读操作反馈。
- 主题切换使用 `eventLabel` 或世界观服务生成的文本，不再直接使用裸地标名。

### Boss

接入位置：

- `js/monster/boss.js`
- `js/game/runDirector.js`

当前接入内容：

- 预警
- 出场
- 打断
- 击退 / 击败
- 扑空
- 命中
- 空投结束

规则：

- Boss 文案必须体现竞技身份，而不是泛用“Boss 出现了”。
- 同一个 Boss 行为类型，在所有入口都要保持同一称呼。

### 结算

接入位置：

- `js/drawer/gameOver.js`

当前接入内容：

- 战报标题
- 模式标签
- 摘要文案
- 身份 / 赛区侧栏信息
- 赛事点评
- 下一目标提示

规则：

- 结算页必须是赛事战报口径。
- 调试局要明确标记不计入联赛成绩和世界观解锁。

## 5. 存档结构

世界观进度保存在 `progress.worldview`。

当前字段：

- `unlockedZones: string[]`
- `unlockedBossProfiles: string[]`
- `lastNarrativeUnlock: string`
- `lastSeasonTier: string`

当前解锁来源：

- 历史最高分达到赛区目标高度
- 本局首次遭遇某类 Boss
- 历史最高分跨过新的联赛席位阈值

规则：

- 世界观进度只存结果，不存临时 UI 文案。
- 任何新解锁逻辑都应写入 `worldview.resolveWorldviewProgress()`，不要散落到多个页面或事件文件里。

## 6. 扩展规则

后续如果把世界观扩展到角色、宠物、商店、排行榜，必须遵守下面的约束：

- 新页面要先定义它在联赛宇宙中的身份，再写文案。
- 新增世界观字段要优先放在 `worldview` 模块或对应配置数据里，不要在页面里临时拼。
- 文案分为两层：
  - 短文本：局内提示、按钮、副标题
  - 中文本：赛区介绍、战报点评、档案说明
- 长文本不要直接塞进高频操作区，尤其是局内和底部操作区。

推荐扩展方向：

- 角色页：选手档案、擅长赛道、联赛称号
- 宠物页：辅助装置 / 伴飞单元设定
- 商店页：成长工坊、联赛补给、赞助模块
- 排行榜：联赛榜单、赛区榜、赛季榜
- 成就页：联赛荣誉、突破纪录、赛区征服

## 7. 修改时的最小检查清单

- 新地标是否同时补齐 `leagueTitle/arenaDesc/bossFactionHint/eventLabel`
- 新 Boss 行为是否补齐对应身份文案
- 主页面、局内、Boss、结算是否仍然使用同一套称呼
- 新世界观文案是否应放进 `worldview` 模块，而不是 UI 文件里硬编码
- 世界观解锁是否只写入 `progress.worldview`
- 调试模式下是否正确降级，不写入联赛进度

## 8. 当前边界

这版世界观模块当前只完成第一阶段：

- 已覆盖：主页面、赛区选择、局内赛段、Boss、结算
- 未覆盖：角色页、宠物页、商店页、排行榜页、成就页的完整联赛化改造

后续开发时，应优先继续沿用“热血竞技联赛”方向，不要再引入另一套平行叙事，否则会重新把页面语义打散。
