# 秀彬跳跳——智能广告投放算法设计

核心目标是：**让每个用户每天看到的广告次数恰到好处——用完额度后，用户“刚刚好还想玩”，保持期待感，而不是“看够了不想玩”或“还有额度没用完就流失”。**

下面我将设计一套**用户分层驱动的动态广告投放算法**，包含用户画像模型、广告配额分配、动态投放策略、以及“恰到好处”的体验控制机制。

---

## 一、算法核心架构

### 1.1 整体框架

```
用户进入游戏
    ↓
【用户画像引擎】→ 识别用户类型（探索型/数值型/社交型/鲸鱼型）
    ↓
【配额分配引擎】→ 计算当日广告总配额（激励视频上限）
    ↓
【实时状态追踪】→ 追踪当日已看次数、当前疲劳度、游戏内进度
    ↓
【动态投放决策】→ 决定是否在该场景展示广告入口
    ↓
【体验控制层】→ 判断“是否已达到今日最佳体验点”
```

### 1.2 核心公式

**当日广告总配额 = 基础额度 × 用户价值系数 × 当日活跃系数 × 疲劳衰减系数**

## 二、用户画像模型（4类用户）

### 2.1 用户分类维度

| 维度 | 指标 | 权重 |
|------|------|------|
| **活跃度** | 近7天登录天数、日均游戏时长 | 30% |
| **广告耐受度** | 日均广告观看次数、主动放弃率、跳过率 | 25% |
| **付费潜力** | 历史付费金额、抽卡次数、装备强化等级 | 25% |
| **游戏偏好** | 主玩模式（跳跃/Boss/抽卡）、职业选择 | 20% |

### 2.2 四类用户画像

| 用户类型 | 占比预估 | 特征 | 广告配额 | 投放策略 |
|---------|---------|------|---------|---------|
| **探索型** | 40% | 新用户、低活跃、未付费、爱尝试不同玩法 | 低（5-8次/日） | 引导式、奖励明确 |
| **数值型** | 30% | 中等活跃、未付费、追求数值成长、爱强化装备 | 中（8-12次/日） | 成长关联、强化加速 |
| **社交型** | 20% | 高活跃、可能付费、爱分享、爱Boss战 | 高（12-18次/日） | 竞争激励、排行榜关联 |
| **鲸鱼型** | 10% | 高活跃、已付费、追求全收集、爱抽卡 | 自定义（15-25次/日） | 价值最大化、去广告优先 |

### 2.3 用户类型识别算法（Python伪代码）

```python
def identify_user_type(user_data):
    """
    输入: user_data包含活跃度、广告耐受度、付费金额、游戏偏好等
    输出: 用户类型
    """
    # 核心判断逻辑
    if user_data['total_paid'] > 100:  # 付费用户优先
        return 'whale'
    
    score_active = user_data['login_days_7d'] * 0.3 + user_data['avg_session_min'] * 0.2
    score_ad_tolerance = user_data['avg_daily_ads'] * 0.4 + (1 - user_data['ad_skip_rate']) * 0.3
    score_growth = user_data['gear_level_avg'] * 0.3 + user_data['char_collection_rate'] * 0.2
    
    if score_active < 0.3:
        return 'explorer'
    elif score_growth > 0.6 and score_ad_tolerance > 0.5:
        return 'grinder'
    elif user_data['share_count_7d'] > 3 or user_data['boss_attempts_7d'] > 10:
        return 'social'
    else:
        return 'explorer'  # 默认
```

## 三、广告配额分配算法

### 3.1 基础配额表

| 用户类型 | 激励视频下限 | 激励视频上限 | 插屏上限 | 总广告配额 |
|---------|------------|------------|---------|-----------|
| 探索型 | 3次 | 8次 | 2次 | 5-10次 |
| 数值型 | 6次 | 12次 | 3次 | 9-15次 |
| 社交型 | 10次 | 18次 | 4次 | 14-22次 |
| 鲸鱼型 | 12次 | 25次 | 2次 | 14-27次 |

### 3.2 动态系数调节

**当日广告总配额 = 基础配额 × 活跃系数 × 疲劳系数 × 付费系数**

#### 活跃系数（基于当日表现）

| 当日活跃表现 | 系数 |
|------------|------|
| 游戏时长<5分钟 | 0.5 |
| 游戏时长5-15分钟 | 0.8 |
| 游戏时长15-30分钟 | 1.0 |
| 游戏时长30-60分钟 | 1.2 |
| 游戏时长>60分钟 | 1.3 |

#### 疲劳系数（基于昨日表现）

| 昨日广告观看 | 疲劳系数 |
|------------|---------|
| 0-5次 | 1.2（还有空间） |
| 6-10次 | 1.0 |
| 11-15次 | 0.8 |
| 16-20次 | 0.6 |
| >20次 | 0.4（需要休息） |

#### 付费系数

| 付费情况 | 系数 |
|---------|------|
| 未付费 | 1.0 |
| 已付费（<50元） | 1.1 |
| 已付费（50-200元） | 1.2 |
| 已付费（>200元） | 1.3 |

### 3.3 配额分配示例

**场景**：数值型用户，昨日看了10次广告，今天已玩20分钟，已付费30元

```
基础配额 = 9次（取中值）
活跃系数 = 1.0（15-30分钟）
疲劳系数 = 1.0（昨日6-10次）
付费系数 = 1.1（已付费30元）

当日总配额 = 9 × 1.0 × 1.0 × 1.1 = 9.9 ≈ 10次
```

## 四、动态投放决策算法

### 4.1 广告点优先级

不同广告点对不同用户类型的价值不同，需要动态调整展示优先级。

| 跳跃复活 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 蓄力加速 | ⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |

| 每日礼包 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |

### 4.2 动态投放决策逻辑

```python
def should_show_ad(ad_type, user_type, current_state, quota_remaining):
    """
    决定是否在某个场景展示广告入口
    """
    # 1. 配额检查
    if quota_remaining <= 0:
        return False
    
    # 2. 冷却检查（同一广告点有最小间隔）
    if not check_cooldown(ad_type, user_id):
        return False
    
    # 3. 疲劳检查（连续广告后需要间隔）
    if current_state['consecutive_ads'] >= 2:
        return False
    
    # 4. 价值评分
    value_score = get_ad_value_score(ad_type, user_type, current_state)
    
    # 5. 用户状态匹配
    user_readiness = calculate_user_readiness(user_type, current_state)
    
    # 6. 综合决策
    trigger_threshold = calculate_threshold(quota_remaining, current_state['session_length'])
    
    return (value_score * user_readiness) > trigger_threshold
```

### 4.3 不同用户类型的投放策略

#### 探索型用户（新用户/低活跃）

**策略**：引导式、低压力、奖励明确

| 投放原则 | 具体做法 |
|---------|---------|
| 早期引导 | 前3天大幅降低广告频率，让用户先爱上游戏 |
| 奖励明确 | 每次广告前明确告知奖励（“看广告得1次免费抽卡”） |
| 位置优先 | 优先展示高价值广告点（免费抽卡、每日礼包） |
| 避免插屏 | 前7天不展示插屏广告 |
| 配额用尽信号 | 弹窗提示：“今日免费福利已领完，明天再来吧~” |

#### 数值型用户（追求成长的免费用户）

**策略**：成长关联、强化加速、高频但可预期

| 投放原则 | 具体做法 |
|---------|---------|
| 强化关联 | 强化加速广告点优先级最高 |
| 批量奖励 | 允许“看3次广告获得全套强化材料” |
| 进度可见 | 显示“今日还可看X次广告获得强化加成” |
| 装备获取 | 装备获取广告与当前装备需求匹配 |
| 配额用尽信号 | “今日强化加速次数已用完，明天继续变强！” |

#### 社交型用户（高活跃、爱竞争）

**策略**：竞争激励、排行榜关联、Boss战强化

| 投放原则 | 具体做法 |
|---------|---------|
| Boss关联 | Boss强化/复活广告点优先级最高 |
| 排行榜 | “观看广告刷新挑战次数，冲击排行榜” |
| 分享激励 | 分享后奖励广告观看次数 |
| 组队加成 | “观看广告为队友提供Boss战加成” |
| 配额用尽信号 | “今日助战次数已用完，明天再来帮助队友！” |

#### 鲸鱼型用户（已付费的高价值用户）

**策略**：价值最大化、去广告优先、尊重付费体验

| 投放原则 | 具体做法 |
|---------|---------|
| 去广告特权 | 付费用户插屏频率降低80% |
| 广告奖励加成 | 激励视频奖励+20%（让付费用户觉得“更值”） |
| 自定义配额 | 允许用户设置每日广告上限 |
| 高价值广告 | 优先展示强化加速、抽卡等高价值点位 |
| 配额用尽信号 | 温柔提示，并推荐内购去广告卡 |

## 五、“恰到好处”体验控制算法

核心目标是：**当用户用完当日配额时，正处于“刚刚好还想玩”的状态**。

### 5.1 体验满意度模型

定义用户在某个时刻的**体验满意度分数**：

```
Satisfaction = (游戏成就达成度 × 0.3) + (资源获取满足度 × 0.2) + (成长进度感知 × 0.3) + (期待感 × 0.2)
```

#### 各维度计算方式

| 维度 | 计算方式 | 理想值 |
|------|---------|-------|
| 游戏成就达成度 | 最近一局高度/个人最高高度 | 80-90% |
| 资源获取满足度 | 当日获取资源/当日预期资源 | 70-80% |
| 成长进度感知 | 当前等级/今日升级预期 | 已完成一个里程碑 |
| 期待感 | 是否有未完成但想完成的目标 | 有1-2个清晰目标 |

### 5.2 配额用尽时机优化

**目标**：让配额用尽时刻的满意度分数达到**0.7-0.85**（“刚刚好”区间）

```python
def optimize_quota_exhaustion(user_type, session_progress, current_satisfaction):
    """
    动态调整剩余配额分配，让用尽时刻落在最优区间
    """
    # 最优满意度区间
    OPTIMAL_SATISFACTION_MIN = 0.7
    OPTIMAL_SATISFACTION_MAX = 0.85
    
    remaining_quota = get_remaining_quota()
    
    # 预测看完剩余配额后的满意度
    predicted_satisfaction = predict_satisfaction_after_ads(
        remaining_quota, session_progress, user_type
    )
    
    if predicted_satisfaction < OPTIMAL_SATISFACTION_MIN:
        # 满意度太低，需要增加高价值广告点
        return 'increase_high_value_ads'
    
    elif predicted_satisfaction > OPTIMAL_SATISFACTION_MAX:
        # 满意度太高（用户可能已经满足，不需要更多广告）
        # 策略：提前用尽配额，让用户带着期待离开
        return 'accelerate_quota_consumption'
    
    else:
        # 恰到好处，保持当前节奏
        return 'maintain_pace'
```

### 5.3 “还想要”信号机制

当用户用完配额后，通过以下信号判断是否达到“刚刚好”状态：

| 信号 | 含义 | 达标判断 |
|------|------|---------|
| 主动退出 | 用户主动关闭游戏 | ❌ 不达标（可能看够了） |
| 后台挂起 | 用户切换到后台 | ✅ 达标（可能还会回来） |
| 查看商店 | 用户去看了内购商品 | ✅✅ 非常达标（想继续玩但不想等广告） |
| 查看排行榜 | 用户查看排行榜 | ✅ 达标（有竞争动力） |
| 分享游戏 | 用户分享游戏 | ✅✅ 非常达标（愿意传播） |
| 次日留存 | 第二天登录 | ✅✅✅ 最优信号 |

## 六、动态投放的实时调控

### 6.1 实时调控参数

| 参数 | 计算方式 | 调控作用 |
|------|---------|---------|
| 会话疲劳度 | 当前会话广告观看次数/该会话时长 | 连续广告超过阈值时降低展示频率 |
| 广告响应率 | 最近5次广告点击/观看率 | 响应率低时降低该类型广告优先级 |
| 游戏内进度 | 当前高度、Boss血量、强化进度 | 关键时刻（濒死/即将升级）提高广告展示优先级 |
| 时段系数 | 根据历史数据，不同时段用户广告意愿 | 高峰时段提高配额 |

### 6.2 场景化动态优先级

```python
def calculate_scene_priority(ad_type, user_state, game_state):
    """
    根据当前游戏状态动态调整广告优先级
    """
    priority = 0.5  # 基础优先级
    
    # 濒死状态（跳跃掉落边缘）
    if game_state['is_near_death'] and ad_type == 'revive':
        priority += 0.4
        
    # 即将升级（强化差一点）
    if game_state['gear_next_level_cost'] < 0.2 and ad_type == 'gear_boost':
        priority += 0.3
        
    # Boss战关键时刻
    if game_state['is_in_boss_fight'] and game_state['boss_hp'] < 0.3:
        if ad_type == 'boss_boost':
            priority += 0.4
        elif ad_type == 'boss_revive':
            priority += 0.3
            
    # 抽卡渴望（刚看到新角色）
    if user_state['just_saw_new_char'] and ad_type == 'free_draw':
        priority += 0.35
        
    return min(priority, 1.0)
```

## 八、总结

这套智能广告投放算法的核心设计原则：

1. **用户分层是基础**：探索型/数值型/社交型/鲸鱼型，不同用户不同策略
2. **动态配额是关键**：当日配额 = 基础 × 活跃系数 × 疲劳系数 × 付费系数
3. **场景化投放是灵魂**：在用户最需要的时候（濒死、即将升级）展示最相关的广告
4. **“恰到好处”是目标**：配额用尽时，用户满意度在0.7-0.85之间，带着期待离开
5. **实时调控是保障**：根据会话疲劳度、响应率、游戏进度动态调整
