# 美术提示词工作流

用于文生图图集的完整工作流：清理目录 → 需求分析 → 比例分组 → 提示词生成 → 图集拆解 → rembg去背景 → 缩放像素。

---

## 阶段零：清理工作目录

**在开始工作流之前，必须先清理 art-assets 目录，确保干净的工作环境。**

执行以下命令清理目录：
```bash
rm -rf D:/GitWechatGame/art-assets/*
```

或者保留目录结构但清空内容：
```bash
# 清理 specs 目录
rm -rf D:/GitWechatGame/art-assets/specs/*
# 清理 prompts 目录
rm -rf D:/GitWechatGame/art-assets/prompts/*
# 清理 atlas 目录
rm -rf D:/GitWechatGame/art-assets/atlas/*
```

**注意：** 清理前请确认是否需要保留已有的图集文件或输出结果。

## 阶段一：需求分析与元素拆解

分析用户需求，识别所有需要的UI元素：
- 元素类型（图标/按钮/面板/头像等）
- 数量和尺寸规格
- 美术风格要求

**输出：元素清单（含中文名称、用途、建议尺寸）**

## 阶段二：按宽高比分组

将相同宽高比（比例）的元素归为一组。

**分组原则：**
- 计算每个元素的宽高比（如64x64=1:1, 128x128=1:1）
- 相同比例的元素归为同一组
- 每组生成独立的图集和配置文件
- 图集尺寸根据元素数量计算（尽量接近正方形）

**常见宽高比分类：**

| 比例 | 示例尺寸 | 典型用途 |
|-----|---------|---------|
| 1:1 | 64x64, 128x128, 256x256 | 图标、头像 |
| 2:1 | 200x100, 256x128 | 横向按钮 |
| 1:2 | 100x200, 128x256 | 竖向面板 |
| 3:1 | 300x100 | 宽条形按钮 |
| 4:3 | 160x120 | 缩略图 |
| 16:9 | 320x180 | 横幅 |

**分组命名规则：**
- `ratio_1_1` - 1:1比例组
- `ratio_2_1` - 2:1比例组
- `ratio_1_2` - 1:2比例组

**示例分组：**
```
分组1 [ratio_1_1]：商店图标(96x96)、奖杯图标(128x128)、头像框(64x64) → 目标输出64x64
分组2 [ratio_2_1]：主按钮(200x100)、确认按钮(256x128) → 目标输出200x100
分组3 [ratio_1_2]：侧边栏(100x200) → 目标输出100x200
```

## 阶段二点五：标准画布与网格规范

**【重要技巧】为保证AI生成图片的比例正确性，所有资源（包括静态资源和帧动画）都应采用此方法：**

### 核心原则

1. **生成 1:1 标准画布**：1024×1024、1536×1536、2048×2048 等
2. **元素居中留白**：元素在格子内居中放置，**元素占据不到整个画面时，用纯白色填充空白区域**
3. **严格控制网格**：确保图片划分的比例和行列精确，保证图片规格正确且连续
4. **适用所有资源**：此技巧**不止限于帧动画**，普通静态资源生成也保持这样

### 标准画布规格表

| 帧数 | 网格布局 | 单元尺寸 | 画布尺寸 | 说明 |
|-----|---------|---------|---------|------|
| 1 | 1×1 | 1024×1024 | 1024×1024 | 单帧 |
| 2 | 1×2 | 512×1024 | 1024×1024 | 两帧横排 |
| 3-4 | 2×2 | 512×512 | 1024×1024 | 四帧方阵 |
| 5-6 | 3×2 | 512×512 | 1536×1024 | 六帧横条 |
| 7-9 | 3×3 | 512×512 | 1536×1536 | 九宫格 |
| 10-12 | 4×3 | 512×512 | 2048×1536 | 十二帧 |

### 宽高比与网格单元对应

不同宽高比的元素，在 512×512 网格内的占位：

| 元素宽高比 | 网格内尺寸 | 定位方式 |
|-----------|-----------|---------|
| 1:1 | 512×512 | 填满 |
| 2:1 | 512×256 | 垂直居中 |
| 3:1 | 480×160 | 垂直居中 |
| 4:1 | 512×128 | 垂直居中 |
| 6:1 | 480×80 | 垂直居中 |
| 8:1 | 512×64 | 垂直居中 |

### 示例：弹射平台动画（4帧，6:1比例）

```
画布：1024×1024
网格：2×2（每格512×512）
每帧内容：480×80（水平居中，垂直居中于格子中央）

┌─────────────┬─────────────┐
│             │             │
│   ───────   │   ───────   │
│  第1帧静止  │  第2帧蓄力  │
│             │             │
├─────────────┼─────────────┤
│             │             │
│   ───────   │   ───────   │
│  第3帧爆发  │  第4帧恢复  │
│     ↑↑↑     │             │
│             │             │
└─────────────┴─────────────┘
```

### JSON配置示例

```json
{
  "canvas": {
    "width": 1024,
    "height": 1024
  },
  "grid": {
    "cols": 2,
    "rows": 2,
    "cell_size": {
      "width": 512,
      "height": 512
    }
  },
  "element_in_cell": {
    "size": {"width": 480, "height": 160},
    "alignment": "center",
    "padding_note": "元素在格子内水平垂直居中，四周留白填充纯白色"
  }
}
```

### 提示词模板

```
请生成一张 1024×1024 像素的游戏UI图集，背景纯白色。

【画布与网格】
- 画布尺寸：1024×1024 像素
- 网格布局：2行2列，共4个格子
- 每个格子：512×512 像素
- 每个格子内的元素尺寸：480×160 像素
- 元素在每个格子内水平垂直居中
- 格子内元素以外的区域填充纯白色

【网格位置说明】
- 左上格：第1帧
- 右上格：第2帧
- 左下格：第3帧
- 右下格：第4帧
```

## 阶段二点五点五：序列帧动画特殊处理

**对于需要序列帧动画的资源（如弹射、崩塌、爆炸等），必须遵循「最大帧尺寸原则」。**

### 核心原则

序列帧动画中，**所有帧必须使用完全相同的裁切尺寸**。尺寸规格应以**所有帧中内容最大的一帧**为基准。

**【关键示例】**
- 某图片本身是 64×64
- 某一帧动画跳跃并发射粒子，整个图片变成 64×128
- **那么其他所有动画帧，也都要是 64×128**（即使它们的内容实际只需要 64×64）

### 为什么需要最大帧尺寸？

1. **统一锚点**：所有帧对齐播放时不会抖动
2. **完整显示**：避免特效超出边界被裁切
3. **引擎兼容**：大多数游戏引擎要求序列帧尺寸一致

### 常见最大帧场景

| 动画类型 | 最大帧 | 原因 |
|---------|-------|------|
| 弹射/喷射 | 爆发态 | 向上喷射的能量拖尾/粒子 |
| 爆炸 | 中段 | 火焰/碎片扩散范围最大 |
| 崩塌 | 碎裂态 | 碎片散落范围超出原物体 |
| 跳跃 | 起跳/落地 | 挤压变形时宽度增加 |
| 攻击 | 蓄力/挥砍 | 武器轨迹/特效延伸 |

### 示例：角色跳跃+粒子动画

| 帧序 | 内容 | 实际内容尺寸 | 备注 |
|-----|------|------------|------|
| 1 | 站立态 | 64×64 | 正常角色 |
| 2 | 蹲下蓄力 | 64×48 | 压扁变形 |
| 3 | **起跳+粒子** | **64×128** | **向上跳跃+发射粒子（最大）** ← 基准 |
| 4 | 空中态 | 64×64 | 飞行中 |
| 5 | 落地 | 64×56 | 落地压扁 |

**配置规格：**
- 原始角色尺寸：64×64
- 最大帧尺寸：64×128（起跳+粒子）
- **统一裁切尺寸：64×128**
- **所有帧均使用 64×128**（即使帧1、4只有64×64内容，也要居中放置在64×128画布中）

### 尺寸计算方法

```
1. 确定所有帧中内容范围最大的那一帧
2. 以该帧的边界框为基准，增加 10-20% 安全边距
3. 所有帧统一使用此尺寸
```

**示例：弹射平台动画**

| 帧序 | 内容 | 实际内容高度 | 备注 |
|-----|------|------------|------|
| 1 | 静止态 | 14px | 仅平台本体 |
| 2 | 蓄力态 | 18px | 轻微压缩+发光 |
| 3 | **爆发态** | **40px** | **向上喷射能量（最大）** ← 基准 |
| 4 | 恢复态 | 20px | 残留光粒 |

**配置规格：**
- 原始平台高度：14px
- 最大帧高度：40px（爆发态含喷射效果）
- **统一裁切高度：48px**（40px + 20%安全边距）
- 所有帧均使用 192×48 尺寸

### JSON 配置示例

```json
{
  "spec_name": "ratio_4_1_boost",
  "ratio": "4:1",
  "animation": {
    "type": "sequence",
    "frame_count": 4,
    "max_frame": {
      "frame_index": 2,
      "description": "爆发态 - 向上喷射能量，高度最大",
      "content_bounds": {"width": 192, "height": 40}
    },
    "unified_size_note": "所有帧统一使用192x48尺寸，以爆发态为基准"
  },
  "target_pixel": {
    "width": 192,
    "height": 48
  }
}
```

### 提示词注意事项

在提示词中需要明确说明：

```
【序列帧规格说明】
- 第3帧（爆发态）有向上喷射的能量拖尾，高度最大
- 所有4帧必须统一使用 384×96 像素的网格尺寸
- 第1、2、4帧的平台本体应垂直居中放置，预留上方空间
- 确保所有帧的平台水平位置对齐，避免播放时抖动
```

## 阶段三：生成规格配置JSON

为每个分组生成独立的JSON配置文件，命名格式：`ratio_{宽比}_{高比}.json`

```json
{
  "spec_name": "ratio_1_1",
  "ratio": "1:1",
  "target_pixel": {
    "width": 64,
    "height": 64
  },
  "element_size": {
    "width": 256,
    "height": 256
  },
  "canvas": {
    "width": 512,
    "height": 512
  },
  "grid": {
    "cols": 2,
    "rows": 2
  },
  "elements": [
    {
      "name": "icon_shop",
      "name_cn": "商店图标",
      "original_size": {"width": 96, "height": 96},
      "grid_pos": {"col": 0, "row": 0}
    },
    {
      "name": "icon_trophy",
      "name_cn": "奖杯图标",
      "original_size": {"width": 128, "height": 128},
      "grid_pos": {"col": 1, "row": 0}
    }
  ],
  "compress": {
    "enabled": true,
    "quality": "high"
  }
}
```

**JSON字段说明：**

| 字段 | 说明 |
|-----|------|
| `spec_name` | 规格名称，格式 `ratio_{宽比}_{高比}` |
| `ratio` | 宽高比，格式 `宽:高` |
| `target_pixel` | **目标输出像素** - 裁切后自动缩放到此尺寸 |
| `element_size` | 图集中每个网格元素的尺寸（用于裁切定位） |
| `canvas` | 整个图集的画布尺寸 |
| `grid` | 网格布局（列数、行数） |
| `elements[].original_size` | 元素的原始设计尺寸（参考用） |
| `compress.enabled` | 是否启用压缩 |
| `compress.quality` | 压缩品质：high/medium/low |

**文件命名规则：**
- `ratio_1_1.json` - 1:1比例组配置
- `ratio_2_1.json` - 2:1比例组配置
- `ratio_16_9.json` - 16:9比例组配置

## 阶段四：生成提示词

为每个分组生成独立的提示词文件，命名格式：`ratio_{宽比}_{高比}_prompt.txt`

```
请生成一张[画布宽]x[画布高]像素的游戏UI图集，背景纯白色。

【整体风格】
[美术风格描述]

【图集布局】
图集分为[行数]行[列数]列，每个元素[元素宽]x[元素高]像素。

---

【元素描述】

第一行，从左到右依次是：

1. [元素外观描述]

2. [元素外观描述]

[更多元素...]

---

【要求】
- 背景纯白色
- 元素按网格排列，边界清晰
- 风格统一
```

**重要原则：**
- 提示词中不包含任何英文文件名
- 不包含坐标数字
- 使用自然语言描述位置
- 说明每个元素内容居中，便于裁切

## 阶段五：图集拆解、去背景与缩放

用户提供图集路径后，根据对应的JSON配置进行处理：

```bash
python <skill-path>/scripts/split_atlas.py <图集路径> <规格JSON路径> [输出目录] [选项]
```

**处理流程（默认）：**
1. **拆图** - 按网格裁切每个元素（保持原始高分辨率）
2. **rembg去背景** - 使用AI模型智能去除背景
3. **缩放像素** - 根据 `target_pixel` 自动缩放到目标尺寸

**选项：**
| 选项 | 说明 |
|------|------|
| `--no-rembg` | 跳过 rembg 去背景步骤 |
| `--no-resize` | 跳过缩放像素步骤（保持原始裁剪尺寸） |

**示例：**
```bash
# 完整流程：拆图 → rembg去背景 → 缩放像素
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/"

# 只拆图和缩放，不去背景
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/" --no-rembg

# 只拆图和去背景，不缩放
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/" --no-resize

# 只拆图，保持原始尺寸
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/" --no-rembg --no-resize

# 批量处理
python scripts/split_atlas.py --batch "atlas/" "specs/" "output/"
```

**依赖：**
- rembg 去背景需要安装：`pip install "rembg[cpu]"`
- 首次使用会自动下载模型文件（约176MB）

## 完整工作流示例

**用户输入：**
> 我需要生成登录界面的UI元素：商店图标(96x96)、奖杯图标(128x128)、好友图标(64x64)、金币图标(128x128)、开始按钮(200x100)、取消按钮(256x128)


**步骤4 - 生成提示词：**
```
prompts/
├── ratio_1_1_prompt.txt
└── ratio_2_1_prompt.txt
```

**步骤5 - 用户生成图集后拆解：**
```bash
# 完整流程：拆图 → rembg去背景 → 缩放像素
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/"
# 输出: icon_shop.png (64x64, 透明背景), icon_trophy.png (64x64, 透明背景), ...
```

## 画布尺寸计算

根据元素数量自动计算合适的画布尺寸：

| 元素数量 | 网格布局 | 画布尺寸 |
|---------|---------|---------|
| 1 | 1x1 | 元素尺寸 |
| 2 | 1x2 | 宽 x 高x2 |
| 3-4 | 2x2 | 宽x2 x 高x2 |
| 5-6 | 2x3 | 宽x2 x 高x3 |
| 7-9 | 3x3 | 宽x3 x 高x3 |
| 10-12 | 3x4 | 宽x3 x 高x4 |

**建议：图集中每个网格元素尺寸 = target_pixel × 4**
- 目标64x64 → 网格256x256
- 目标128x128 → 网格512x512

## 宽高比计算工具

```python
from math import gcd

def get_ratio(width, height):
    """计算宽高比"""
    d = gcd(width, height)
    return f"{width//d}:{height//d}"

def get_spec_name(width, height):
    """生成规格名称"""
    d = gcd(width, height)
    return f"ratio_{width//d}_{height//d}"

# 示例
get_ratio(96, 96)    # "1:1"
get_ratio(200, 100)  # "2:1"
get_spec_name(128, 128)  # "ratio_1_1"
```
