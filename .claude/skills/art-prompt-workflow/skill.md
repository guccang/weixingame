# 美术提示词工作流

用于文生图图集的完整工作流：清理目录 → **风格探索与选择** → 需求分析 → **画布规划（正方形+严格网格）** → 提示词生成 → 图集拆解 → rembg去背景 → 缩放像素。

---

## 阶段零：清理工作目录

**在开始工作流之前，必须先清理 art-assets 目录，确保干净的工作环境。**

执行以下命令清理目录：
```bash
rm -rf D:/GitWechatGame/art-assets/*
```

或者保留目录结构但清空内容：
```bash
# 清理 specs 目录
rm -rf D:/GitWechatGame/art-assets/specs/*
# 清理 prompts 目录
rm -rf D:/GitWechatGame/art-assets/prompts/*
# 清理 atlas 目录
rm -rf D:/GitWechatGame/art-assets/atlas/*
```

**注意：** 清理前请确认是否需要保留已有的图集文件或输出结果。

## 阶段零点五：风格探索与选择（交互式）

**此阶段是交互式的，通过多轮对话帮助用户明确想要的美术风格。**

### 第一步：需求分析与风格发散

当用户提出设计需求时，首先分析需求关键词，然后**发散推荐3-5种适合的美术风格方向**。

**分析维度：**
- 游戏类型（休闲/动作/策略/RPG/竞技等）
- 主题氛围（奇幻/科幻/古风/现代/卡通/暗黑等）
- 目标受众（全年龄/儿童/青少年/成人）
- 参考游戏（如果用户有提到）

**风格推荐格式：**

```
根据您的需求，我为您推荐以下几种美术风格：

【风格A】{风格名称}
- 描述：{一句话描述风格特点}
- 特点：{列举2-3个视觉特征}
- 适合场景：{适合的游戏类型或氛围}
- 代表游戏：{1-2个参考游戏}

【风格B】{风格名称}
...

【风格C】{风格名称}
...

请选择一个您喜欢的风格方向（回复A/B/C），或告诉我您想要的其他风格。
```

### 第二步：风格细化

用户选择风格后，进一步细化该风格的子选项：

**常见细化维度：**

| 维度 | 选项示例 |
|------|---------|
| **色调** | 鲜艳明亮 / 柔和淡雅 / 深沉暗黑 / 复古怀旧 |
| **质感** | 扁平简约 / 立体质感 / 手绘风 / 像素风 / 水彩风 |
| **细节** | 极简主义 / 适度细节 / 精细复杂 |
| **光影** | 平光 / 柔和阴影 / 强烈光影 / 发光特效 |
| **轮廓** | 粗边框 / 细边框 / 无边框 / 描边+发光 |

**细化问题格式：**

```
您选择了【{风格名称}】，接下来我们细化一下风格细节：

1. **色调偏好**：
   A. 鲜艳明亮（饱和度高，活力感强）
   B. 柔和淡雅（低饱和，温馨舒适）
   C. 深沉暗黑（暗色调，神秘感）
   D. 复古怀旧（泛黄/褪色效果）

2. **质感偏好**：
   A. 扁平简约（无阴影，色块清晰）
   B. 立体质感（有光影，3D感）
   C. 手绘风格（笔触可见，有机感）
   D. 精致渲染（高度精细，插画感）

请分别回复您的选择，如"1A 2B"或直接描述您的偏好。
```

### 第三步：生成风格描述段落

根据用户的选择，生成完整的**美术风格描述段落**，用于后续提示词：

**风格描述模板：**

```
【美术风格】
风格定位：{主风格}，{子风格特征}
色调：{色调描述}
质感：{质感描述}
细节：{细节程度描述}
光影：{光影处理方式}
轮廓：{边框处理方式}
整体氛围：{一句话总结视觉感受}
```

**示例输出：**

```
【美术风格】
风格定位：卡通风格，扁平化设计，现代简约
色调：鲜艳明亮，饱和度高，色彩活泼
质感：扁平简约，色块清晰，无复杂纹理
细节：适度细节，形状简洁但不失识别度
光影：平光为主，轻微阴影增加层次
轮廓：细边框，部分元素带发光效果
整体氛围：轻松活泼，适合休闲小游戏，给人愉悦感
```

## 阶段二：画布规划与比例分组

**核心目标：画布偏向正方形（适合nanoBanana生成），元素严格按网格排布，切图后直接得到目标比例。**

### 2.1 画布约束条件

| 约束项 | 规则 |
|-------|------|
| **画布形状** | 偏向正方形（适合nanoBanana生成），尺寸范围 512×512 ~ 4096×4096 |
| **最大网格布局** | 4行 × 4列（最多16个元素） |
| **元素数量超限** | 超过16个元素时，拆分成多张画布 |
| **网格排布** | 元素严格按网格排列，每个格子就是元素的实际尺寸，无需居中 |
| **空白填充** | 未填满的格子用纯白色填充 |
| **比例分组** | 相同宽高比的元素归入同一画布 |

### 2.2 画布尺寸计算（偏向正方形 + 严格网格）

**核心原则：选择最接近正方形的网格布局，每个格子尺寸 = 元素实际尺寸。**

```
1. 根据元素数量选择最接近正方形的网格布局
2. 每格尺寸 = 元素实际尺寸（按比例）
3. 画布尺寸 = 每格尺寸 × 网格数
```

#### 示例：4个 4:1 元素（目标192×48）

```
元素尺寸：512×128（宽×高，4:1比例）
网格布局：2列2行（最接近正方形）
画布尺寸：1024×512

┌─────────────────────────┬─────────────────────────┐
│        元素1 512×128    │        元素2 512×128    │
├─────────────────────────┼─────────────────────────┤
│        元素3 512×128    │        元素4 512×128    │
└─────────────────────────┴─────────────────────────┘
           1024px                  512px
```

#### 示例：3个 4:1 元素（目标192×48）

```
元素尺寸：512×128
网格布局：3列1行（或1列3行，选择更接近正方形的）
画布尺寸：1536×128（或512×384，选择后者更接近正方形）

┌───────────────┐
│   元素1       │ 128px
├───────────────┤
│   元素2       │ 128px
├───────────────┤
│   元素3       │ 128px
└───────────────┘
     512px

选择 1列3行布局（512×384），更接近正方形
```

#### 示例：5个 1:1 元素（每个 512×512）

```
元素尺寸：512×512
网格布局：3列2行（6格，第6格填充白色）
画布尺寸：1536×1024

┌─────────┬─────────┬─────────┐
│  元素1  │  元素2  │  元素3  │
├─────────┼─────────┼─────────┤
│  元素4  │  元素5  │  白色   │
└─────────┴─────────┴─────────┘
```

### 2.3 常见比例元素尺寸参考

**推荐：元素尺寸 = target_pixel × 4 或 × 8**

| 目标像素 | 推荐元素尺寸 | 适用场景 |
|---------|-------------|---------|
| 64×64 | 256×256 或 512×512 | 小图标 |
| 128×128 | 512×512 | 中等图标 |
| 192×48 (4:1) | 512×128 | 横长平台 |
| 64×256 (1:4) | 256×1024 | 纵长元素 |

### 2.4 网格布局优先选择（偏向正方形）

| 元素数量 | 网格布局 | 说明 |
|---------|---------|------|
| 1 | 1×1 | 单元素，正方形画布 |
| 2 | 2×1 或 1×2 | 根据元素比例选择更接近正方形的 |
| 3 | 3×1 或 1×3 | 选择更接近正方形的方向 |
| 4 | 2×2 | 正方形网格 |
| 5-6 | 3×2 或 2×3 | 选择更接近正方形的 |
| 7-9 | 3×3 | 正方形网格 |
| 10-12 | 4×3 或 3×4 | 选择更接近正方形的 |
| 13-16 | 4×4 | 正方形网格 |

## 阶段三：序列帧动画特殊处理

**对于需要序列帧动画的资源（如弹射、崩塌、爆炸等），必须遵循「最大帧尺寸原则」和「帧连贯性原则」。**

### 核心原则

#### 原则一：最大帧尺寸
序列帧动画中，**所有帧必须使用完全相同的裁切尺寸**。尺寸规格应以**所有帧中内容最大的一帧**为基准。

**【关键示例】**
- 某图片本身是 64×64
- 某一帧动画跳跃并发射粒子，整个图片变成 64×128
- **那么其他所有动画帧，也都要是 64×128**（即使它们的内容实际只需要 64×64）

#### 原则二：帧连贯性
序列帧动画必须确保**帧与帧之间的视觉连贯性**，避免跳跃或不自然的过渡。

**【帧连贯性要求】**
1. **运动轨迹连续**：相邻帧之间的动作变化要平滑，不能有突变
2. **形态变化渐进**：形状、大小、颜色等变化要逐步过渡
3. **特效衔接自然**：光效、粒子、能量等特效要有产生→持续→消散的完整过程
4. **时间节奏均匀**：每帧的"时间间隔"应该相等，动作幅度均匀

**【连贯性检查清单】**
- ✅ 相邻帧之间动作变化是否平滑（不是跳跃式变化）？
- ✅ 主体位置是否保持相对稳定（除非是移动动画）？
- ✅ 颜色/亮度变化是否渐进？
- ✅ 特效（发光、粒子）是否有明确的增减过程？

### 常见最大帧场景

| 动画类型 | 最大帧 | 原因 |
|---------|-------|------|
| 弹射/喷射 | 爆发态 | 向上喷射的能量拖尾/粒子 |
| 爆炸 | 中段 | 火焰/碎片扩散范围最大 |
| 崩塌 | 碎裂态 | 碎片散落范围超出原物体 |
| 跳跃 | 起跳/落地 | 挤压变形时宽度增加 |
| 攻击 | 蓄力/挥砍 | 武器轨迹/特效延伸 |

### 尺寸计算方法

```
1. 确定所有帧中内容范围最大的那一帧
2. 以该帧的边界框为基准，增加 10-20% 安全边距
3. 所有帧统一使用此尺寸
```

### 帧描述规范

在描述序列帧时，必须使用**连续动作描述法**：

**【错误示例】❌**
```
帧1：平台静止
帧2：平台发光
帧3：平台喷射能量
帧4：平台恢复
```
↑ 问题：描述太独立，缺乏连贯性信息

**【正确示例】✅**
```
这是一个4帧的弹射平台动画，从静止到爆发再到恢复的完整过程：

帧1（起始态）：平台处于静止状态，表面纹理清晰，无明显特效
帧2（蓄力态，从帧1渐进）：平台轻微压缩下陷（压缩幅度约15%），
     表面开始泛起淡蓝色光芒，光晕从中心向外扩散
帧3（爆发态，从帧2渐进）：光芒达到最亮，向上喷射出3道能量光束，
     平台略微回弹，光束带有粒子拖尾效果
帧4（恢复态，从帧3渐进）：光束逐渐消散，粒子效果减弱，
     平台恢复原始形态，残留少量光晕渐隐
```
↑ 优点：明确标注了帧之间的渐进关系，描述了完整的动作弧线

## 阶段六：图集拆解、去背景与缩放

用户提供图集路径后，根据对应的JSON配置进行处理：

```bash
python <skill-path>/scripts/split_atlas.py <图集路径> <规格JSON路径> [输出目录] [选项]
```

**处理流程（默认）：**
1. **拆图** - 按网格裁切每个元素（保持原始高分辨率）
2. **rembg去背景** - 使用AI模型智能去除背景
3. **缩放像素** - 根据 `target_pixel` 自动缩放到目标尺寸

**选项：**
| 选项 | 说明 |
|------|------|
| `--no-rembg` | 跳过 rembg 去背景步骤 |
| `--no-resize` | 跳过缩放像素步骤（保持原始裁剪尺寸） |

**示例：**
```bash
# 完整流程：拆图 → rembg去背景 → 缩放像素
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/"

# 只拆图和缩放，不去背景
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/" --no-rembg

# 只拆图和去背景，不缩放
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/" --no-resize

# 只拆图，保持原始尺寸
python scripts/split_atlas.py "atlas/ratio_1_1.png" "specs/ratio_1_1.json" "output/" --no-rembg --no-resize

# 批量处理
python scripts/split_atlas.py --batch "atlas/" "specs/" "output/"
```

**依赖：**
- rembg 去背景需要安装：`pip install "rembg[cpu]"`
- 首次使用会自动下载模型文件（约176MB）

## 宽高比计算工具

```python
from math import gcd

def get_ratio(width, height):
    """计算宽高比"""
    d = gcd(width, height)
    return f"{width//d}:{height//d}"

def get_spec_name(width, height):
    """生成规格名称"""
    d = gcd(width, height)
    return f"ratio_{width//d}_{height//d}"

# 示例
get_ratio(96, 96)    # "1:1"
get_ratio(200, 100)  # "2:1"
get_spec_name(128, 128)  # "ratio_1_1"
```
