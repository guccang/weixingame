# 游戏角色序列帧提示词生成器

生成用于 AI 文生图工具的游戏角色序列帧动画图集提示词。

---

## 工作流程

1. 收集用户需求（角色形象、动画类型、画布参数）
2. 确定网格布局与画布尺寸
3. 生成完整的文生图提示词
4. 保存提示词到 `art-assets/prompts/` 目录

---

## 第一步：收集需求

向用户确认以下信息（如果用户已提供则跳过）：

### 必需信息

| 项目 | 说明 | 示例 |
|------|------|------|
| **角色形象** | 角色外观描述 | 钢铁侠、骑士、忍者、小动物等 |
| **动画类型** | 需要什么动画 | 跳跃、奔跑、攻击、待机、受伤、死亡 |
| **帧数** | 动画帧数量 | 通常 4-8 帧 |

### 可选信息（有默认值）

| 项目 | 默认值 | 说明 |
|------|--------|------|
| **美术风格** | 卡通风格，Q版可爱 | 整体视觉风格 |
| **视角** | 侧面视角（面向右） | 角色朝向 |
| **画布尺寸** | 自动计算 | 可由用户指定 |
| **每格尺寸** | 512×512 | 单帧尺寸 |

### 角色形象引导

如果用户只说了角色类型（如"一个战士"），帮助细化：

```
请告诉我更多关于角色外观的细节：
- 性别/体型
- 服装/装备（盔甲、武器、帽子等）
- 特征色/主色调
- 有无特殊效果（发光、火焰、光环等）
```

---

## 第二步：确定画布与网格

### 网格布局规则

根据帧数选择最接近正方形的网格布局：

| 帧数 | 网格布局 | 画布尺寸（每格512） |
|------|---------|-------------------|
| 4 | 2×2 | 1024×1024 |
| 5 | 3×2 | 1536×1024 |
| 6 | 3×2 | 1536×1024 |
| 7-8 | 4×2 | 2048×1024 |
| 9 | 3×3 | 1536×1536 |
| 10-12 | 4×3 | 2048×1536 |

如果用户指定了不同的每格尺寸，画布 = 每格尺寸 × 网格数。

### 画布计算示例

```
6帧，每格512×512 → 3列2行 → 1536×1024
但用户要求 1024×1536（3行2列），也可以 — 以用户需求为准
```

---

## 第三步：生成提示词

### 提示词结构

提示词必须包含以下部分，按顺序排列：

```
【画布与网格】
【整体风格】
【重要约束 - 无Y轴位移】
【序列帧描述】
【要求】
```

### 提示词模板

```
请生成一张 [画布宽]×[画布高] 像素的游戏角色序列帧图集，背景纯白色。

【画布与网格】
- 画布尺寸：[画布宽]×[画布高] 像素
- 网格布局：[行数]行[列数]列，共[帧数]个格子
- 每个格子：[格宽]×[格高] 像素
- 角色在每个格子内水平垂直居中
- 格子内角色以外的区域填充纯白色

【整体风格】
- [风格描述，如：卡通风格，适合微信跳一跳小游戏]
- [视角，如：侧面视角（角色面向右侧）]
- 线条清晰，色彩鲜明
- [比例，如：可爱Q版比例，头大身小]
- [角色详细外观描述]

【重要约束 - 无Y轴位移】
- 所有[帧数]帧中，角色的垂直位置必须完全一致
- 角色脚底在每帧中都在同一水平线上
- 只有姿态变化，没有高度变化（高度变化由游戏代码控制）
- 确保角色在各帧中的水平位置也保持一致

【序列帧描述】

[按行描述每帧，使用下面的帧描述模板]

【要求】
- 背景纯白色，便于后期去背景
- 每帧角色大小一致，约占据格子的60-70%
- 角色轮廓清晰，边缘干净
- [帧数]帧动作连贯，形成流畅的[动画类型]动画
- 确保所有帧中角色的垂直和水平位置对齐，避免播放时抖动
- [特效递进描述，如适用]
```

### 帧描述方法

每一帧必须包含：**姿态名称 + 身体各部位状态 + 特效状态**。

**关键原则：帧之间用"从帧X渐进过渡"标注连贯性。**

```
第X行（从左到右）：

N. [姿态名称]：[角色名][具体姿态描述]，[腿部状态]，[臂部状态]，[特效/发光状态]，[整体感觉词]
```

## 预设动画模板

以下是常见动画类型的帧预设。根据用户需求选用，并在其基础上根据角色形象定制描述。

### 跳跃动画（6帧）

| 帧 | 姿态 | 身体状态 | 特效 |
|----|------|---------|------|
| 1 | 站立 | 双腿直立，双臂下垂 | 基础状态 |
| 2 | 蓄力 | 蹲下，双腿弯曲，身体压扁变宽 | 蓄力特效开始 |
| 3 | 起跳 | 身体舒展，双腿蹬直 | 推进器/特效喷射 |
| 4 | 上升 | 身体紧凑，空中姿态 | 推进器/特效持续 |
| 5 | 下落 | 身体略微前倾，准备落地 | 特效减速 |
| 6 | 落地 | 双腿弯曲缓冲，身体压扁 | 特效熄灭 |

### 奔跑动画（6帧）

| 帧 | 姿态 | 身体状态 | 特效 |
|----|------|---------|------|
| 1 | 起跑 | 前腿弓步，后腿蹬直，前倾 | 起始 |
| 2 | 前跨 | 前腿伸出，后腿收起，前摆臂 | 拖尾开始 |
| 3 | 腾空 | 双腿离地，最大步幅 | 拖尾最强 |
| 4 | 落地 | 前腿落地支撑 | 拖尾减弱 |
| 5 | 后蹬 | 后腿发力蹬地 | 拖尾减弱 |
| 6 | 收步 | 双腿收拢准备下一轮 | 拖尾消失 |

### 攻击动画（6帧）

| 帧 | 姿态 | 身体状态 | 特效 |
|----|------|---------|------|
| 1 | 待机 | 正常站立，手持武器 | 无 |
| 2 | 蓄力 | 身体后仰，武器蓄力 | 能量聚集 |
| 3 | 前挥 | 向前挥出武器 | 能量释放 |
| 4 | 命中 | 武器到达最远点 | 冲击波/特效最强 |
| 5 | 收招 | 武器回收 | 特效消散 |
| 6 | 恢复 | 回到待机姿态 | 残余特效消失 |

### 待机/呼吸动画（4帧）

| 帧 | 姿态 | 身体状态 | 特效 |
|----|------|---------|------|
| 1 | 正常 | 自然站立 | 无 |
| 2 | 吸气 | 身体微微上抬，胸腔扩张 | 轻微发光 |
| 3 | 呼气 | 身体微微下沉 | 发光保持 |
| 4 | 恢复 | 回到正常姿态 | 发光消失 |

### 受伤动画（4帧）

| 帧 | 姿态 | 身体状态 | 特效 |
|----|------|---------|------|
| 1 | 正常 | 自然站立 | 无 |
| 2 | 受击 | 身体后仰，表情痛苦 | 受击闪光 |
| 3 | 踉跄 | 前倾不稳 | 闪红减弱 |
| 4 | 恢复 | 摇晃站稳 | 恢复正常色 |

## 完整示例

**用户输入：**
> 生成钢铁侠跳跃序列帧，6帧，1024×1536

**生成的提示词：**

```
请生成一张 1024×1536 像素的游戏角色序列帧图集，背景纯白色。

【画布与网格】
- 画布尺寸：1024×1536 像素
- 网格布局：3行2列，共6个格子
- 每个格子：512×512 像素
- 角色在每个格子内水平垂直居中
- 格子内角色以外的区域填充纯白色

【整体风格】
- 卡通风格，适合微信跳一跳小游戏
- 侧面视角（角色面向右侧）
- 线条清晰，色彩鲜明
- 可爱Q版比例，头大身小
- 钢铁侠形象：红金色盔甲，Q版可爱风格，头盔面罩，胸口反应器发光，手部和脚部推进器

【重要约束 - 无Y轴位移】
- 所有6帧中，角色的垂直位置必须完全一致
- 角色脚底在每帧中都在同一水平线上
- 只有姿态变化，没有高度变化（高度变化由游戏代码控制）
- 确保角色在各帧中的水平位置也保持一致

【序列帧描述】

第一行（从左到右）：
1. 站立姿态：钢铁侠正常站立，双腿直立，双臂自然下垂，胸口反应器发出蓝白色光芒，姿态威武
2. 蓄力姿态：钢铁侠蹲下，双腿弯曲，身体压扁变宽，重心降低，脚部推进器开始发光，准备发射

第二行（从左到右）：
3. 起跳姿态：钢铁侠身体向上舒展，双腿蹬直，脚部推进器喷射火焰，双臂向后，姿态动感
4. 上升姿态：钢铁侠在空中，身体紧凑，脚部和手部推进器持续喷射火焰，飞行姿态帅气

第三行（从左到右）：
5. 下落姿态：钢铁侠身体略微前倾，推进器减速，双腿准备落地，姿态从容
6. 落地姿态：钢铁侠落地瞬间，双腿弯曲缓冲，身体压扁变宽，推进器熄灭，姿态稳重

【要求】
- 背景纯白色，便于后期去背景
- 每帧角色大小一致，约占据格子的60-70%
- 角色轮廓清晰，边缘干净
- 6帧动作连贯，形成流畅的跳跃动画
- 确保所有帧中角色的垂直和水平位置对齐，避免播放时抖动
- 钢铁侠的推进器火焰效果在蓄力、起跳、上升帧中逐渐增强，下落和落地帧中逐渐熄灭
```

## 保存提示词

生成提示词后，保存到项目目录：

```
art-assets/prompts/[描述性名称]_prompt.txt
```

例如：
- `art-assets/prompts/ironman_jump_6f_prompt.txt`
- `art-assets/prompts/knight_attack_6f_prompt.txt`
- `art-assets/prompts/ninja_run_6f_prompt.txt`

## 与 art-prompt-workflow 的区别

| 特性 | 本 skill | art-prompt-workflow |
|------|---------|-------------------|
| 专注领域 | 角色序列帧动画 | UI图集、图标等通用美术资源 |
| 关键约束 | 无Y轴位移、帧间连贯性 | 网格排布、比例分组 |
| 后续处理 | 配合 img-split-rembg-compress 拆解 | 内置拆解去背景流程 |
| 动画预设 | 提供跳跃/奔跑/攻击等模板 | 无 |
